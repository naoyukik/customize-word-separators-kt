# Word Parser Patterns Reference

`WordParser` で使用される標準的な正規表現パターンの定義と、日本語特有の文字種判定の仕様をまとめる。

## 標準文字種パターン

| 種別           | 正規表現                                         | 範囲           |
|:-------------|:---------------------------------------------|:-------------|
| `cjk`        | `[\u3400-\u9FFF\uF900-\uFAFF]+`              | 漢字（CJK 統合漢字） |
| `hiragana`   | `[\u3040-\u309F]+`                           | ひらがな         |
| `katakana`   | `[\u30A1-\u30FA\u30FC-\u30FE]+`              | カタカナ         |
| `kanaSymbol` | `[\u30A0\u30FB]+`                            | カタカナの記号      |
| `latin`      | `[\u0030-\u0039\u0041-\u005A\u0061-\u007A]+` | 英数字（半角）      |

## パースロジックの改善方針（リファクタリング対象）

1. **正規表現の事前コンパイル**: 現在、`wordParse` 呼び出しごとに `Regex` が構築されているが、これを `Lazy`
   または定数として保持し、パフォーマンスを改善する。
2. **カスタムパターンの柔軟性**: ユーザーが設定したカスタムパターンを標準パターンの前に挿入し、優先順位を制御可能にする。
3. **エッジケースのバリデーション**: カンマ区切りの設定データのパースエラーに対する堅牢性を向上させる。
