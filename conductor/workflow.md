# Workflow Configuration

## Core Principles
- **TDD (Test-Driven Development):** 可能な限りテストを先行させ、品質を保証する。
- **Surgical Changes:** 不要な変更を避け、タスクに集中した実装を行う。

## Requirements
- **Test Coverage:** >80% (Kover 報告に基づく)
- **Commit Frequency:** Per Task (タスク完了ごとにコミット)
- **Summary Storage:** Commit Messages (要約をコミットメッセージ本文に含める)

## Phase Completion Verification
- 各フェーズ（Research, Strategy, Execution, Validation）完了時にチェックポイントを設け、ユーザー承認を得る。
- **User Manual Verification:** フェーズ完了時にプロトコルに従い、手動での動作確認を行う。
